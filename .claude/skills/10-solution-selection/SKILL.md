# 솔루션 선정

## 목적

아이디어를 체계적으로 평가하고 최적의 솔루션을 선택합니다.

## 사용 시점

- 아이디어 발상이 완료된 후
- 비즈니스 모델 설계 전
- 여러 솔루션 중 최적안을 선택해야 할 때
- 사용자가 "솔루션 선정", "아이디어 평가", "우선순위"를 언급할 때

## 필수 입력
- 아이디어 발상 결과 (ideation 결과):
  - `think/솔루션탐색.md`
  - `think/솔루션후보.md`
- 문제 가설: `define/문제가설.md` (problem-hypothesis 결과)
- 방향성 정의 (direction-setting 결과):
  - `think/킹핀문제.md`
  - `think/문제해결방향성.md`

## 평가 프레임워크

### 1. ICE 스코어링 프레임워크

각 아이디어를 1-10 척도로 평가:
- **I**mpact (임팩트): 문제 해결 효과성
- **C**onfidence (확신): 성공 가능성
- **E**ase (용이성): 구현 난이도 (역수)

**ICE 점수** = (Impact + Confidence + Ease) / 3

### 2. RICE 스코어링 프레임워크 (선택사항)

- **R**each (도달): 영향받는 사용자 수
- **I**mpact (임팩트): 1인당 영향
- **C**onfidence (확신): 데이터 신뢰도
- **E**ffort (노력): 필요한 인월(person-months)

**RICE 점수** = (Reach × Impact × Confidence) / Effort

## 평가 가이드라인

- 평가는 객관적이고 데이터 기반이어야 함
- 선정된 솔루션은 MVP 구현 가능해야 함
- 단기 실현 가능성과 장기 잠재력 모두 고려
- 혁신과 실현 가능성의 균형
- 고객 니즈와의 정렬 확인
- 명확한 성공 기준 정의

## 도구 활용

### Sequential MCP 사용
복잡한 평가와 의사결정이 필요할 때 Sequential MCP를 활용하여 체계적으로 최적 솔루션을 도출하세요.

## 주의사항

- ICE 점수는 객관적 데이터 기반
- MVP 범위는 3개월 내 개발 가능하게
- 기술 스택은 팀 역량 고려
- KPI는 측정 가능하고 구체적으로
- 리스크는 완화 전략과 함께
- 선정 근거는 명확하고 설득력 있게
