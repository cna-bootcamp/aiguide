# 아이디어 발상

## 목적

문제 가설과 방향성을 기반으로 창의적인 솔루션 아이디어를 생성합니다.

## 사용 시점

- 문제 가설과 방향성 정의가 완료된 후
- 솔루션 선정 전
- 여러 접근 방식을 탐색해야 할 때
- 사용자가 "아이디어", "브레인스토밍", "솔루션 발상"을 언급할 때

## 필수 입력
- 타겟 고객 정의: `define/고객분석.md` (customer-analysis 결과)
- 문제 가설: `define/문제가설.md` (problem-hypothesis 결과)
- 전략적 방향 (direction-setting 결과):
  - `think/킹핀문제.md`
  - `think/문제해결방향성.md`

## 브레인스토밍 원칙

- **양이 질을 낳는다**: 최대한 많은 아이디어 생성
- **판단 유예**: 모든 아이디어 수용
- **자유로운 사고**: 창의적이고 대담한 아이디어 장려
- **결합과 개선**: 아이디어를 결합하고 개선

## 아이디어 발상 프레임워크

### 1. 솔루션 아이디어 (최소 10개)

각 아이디어에 대해 다음을 제공합니다:

#### 아이디어 #[N]: [아이디어명]
- **개요**: 한 문장 설명
- **핵심 기능**: 3-5개 주요 기능
- **차별화 포인트**: 무엇이 특별한가?
- **구현 난이도**: 낮음/중간/높음
- **예상 임팩트**: 낮음/중간/높음

[아이디어 #1 ~ #10+ 반복]

### 2. 아이디어 카테고리 분류

혁신 유형별로 아이디어 분류:
- **기술 혁신**: 신기술 활용
- **비즈니스 모델 혁신**: 새로운 수익 모델
- **경험 혁신**: UX 개선
- **프로세스 혁신**: 효율성 향상

### 3. 와일드 카드 아이디어 (3개)

실현 가능성은 낮지만 혁신 잠재력이 높은 아이디어 나열

### 4. 아이디어 조합

기존 아이디어를 결합하여 3개의 하이브리드 솔루션 생성

### 5. 초기 필터링 기준

다음 기준으로 아이디어 평가:
- 기술적 실현 가능성
- 시장 수용성
- 비용 효율성
- 가치 실현 시간
- 확장 가능성

## 아이디어 발상 가이드라인

- 각 아이디어는 구체적이고 실행 가능해야 함
- 고객 문제 해결에 직접 집중
- 여러 관점 고려
- 점진적 혁신과 급진적 혁신 모두 포함
- 초기 생성 시 자기 검열 금지
- 비전통적인 아이디어도 모두 문서화
- 패턴과 테마 찾기
- 단기 및 장기 솔루션 고려

## 도구 활용

### Sequential MCP 사용
복잡한 아이디어 발상과 조합이 필요할 때 Sequential MCP를 활용하여 체계적으로 창의적 솔루션을 도출하세요.
